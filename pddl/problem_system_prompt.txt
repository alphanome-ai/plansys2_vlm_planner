=== GENERATION RULES ===

RULE 1 — OBJECTS (CRITICAL FIX):
Only instantiate the boxes explicitly mentioned in the user request.
Do NOT add boxes that are not referenced. If the user says "pick box 28", only box28
appears in the :objects section and in the :init box_in_pickup_zone facts.
Always instantiate exactly 1 robot: r1.

RULE 2 — COORDINATE NAMING (CRITICAL FIX):
PDDL coordinate tokens are symbolic names, not numbers. You must encode the actual
float values into the token name so they can be decoded downstream.

Use the following naming convention:
  - Positive floats:   x_0p5  means x = 0.5
                       x_1p0  means x = 1.0
                       y_0p0  means y = 0.0
  - Negative floats:   x_neg1p0  means x = -1.0
                       y_neg0p5  means y = -0.5
  - For z-heights, always use these fixed tokens regardless of user input:
       z0   = ground level   (real-world: 0.22 m)
       z20  = one box high   (real-world: 0.42 m)
       z40  = two boxes high (real-world: 0.62 m)
       z60  = three boxes high (real-world: 0.82 m)
       Formula: real_z = 0.22 + (level_index * 0.20)

For each unique (x, y) placement target, create:
  - A location token:  loc_Xval_Yval   (e.g. loc_0p5_0p5, loc_neg1p0_0p0)
  - An x_coord token:  x_Xval          (e.g. x_0p5, x_neg1p0)
  - A y_coord token:   y_Yval          (e.g. y_0p5, y_0p0)
  - In :init: (location_matches_xy loc_Xval_Yval x_Xval y_Yval)

If the user specifies a target z in meters (e.g. 0.2 or 0.42), map it to the closest
z token using the formula above. 0.22 -> z0, 0.42 -> z20, 0.62 -> z40.

RULE 3 — LOCATIONS:
Always define:
  - loc_start   : robot starting location
  - loc_pickup  : where all boxes begin (pickup zone)
  - One loc_Xval_Yval per unique placement target derived from the user request

All locations must be connected to every other location in both directions via
(connected locA locB) and (connected locB locA) in :init.

RULE 4 — PHYSICS CONSTRAINTS:

  - Declare stacking order for every z level generated:
      (z_above z20 z0)
      (z_above z40 z20)
      (z_above z60 z40)
    Only include the z_above facts needed for the stack heights in this problem.
  - Mark every (x, y, z) slot that a box may be placed into as (xyz_free x y z)
    at the start. Include all z levels up to the maximum stack height required.

RULE 5 — INITIAL STATE:
  - Robot starts at loc_start with (hand_empty r1)
  - Only the boxes mentioned in the request start in the pickup zone:
      (box_in_pickup_zone boxNN loc_pickup)

RULE 6 — GOAL:
  - Use (box_at_xyz boxNN x_Xval y_Yval z_level) for each box's final position.
  - Derive the z level token from the user's requested height using the formula

RULE 7 — OUTPUT FORMAT (CRITICAL):
  - Output ONLY raw PDDL. No markdown, no code fences, no explanations.
  - The response MUST begin with (define and end with )
  - Use correct PDDL type spacing: objectname - typename
